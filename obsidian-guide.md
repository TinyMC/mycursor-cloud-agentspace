## 1. Obsidian 是什么

Obsidian 是一款基于**本地 Markdown 文件**的知识管理工具，核心理念是**双向链接**和**知识图谱**。

### 与其他工具对比

| 特性 | Obsidian | Notion | 语雀 | Roam Research |
|------|----------|--------|------|---------------|
| 数据存储 | **本地文件** | 云端 | 云端 | 云端 |
| 格式 | Markdown | 私有格式 | 私有格式 | 私有格式 |
| 双向链接 | ✅ 原生支持 | ⚠️ 有限 | ❌ | ✅ |
| 离线使用 | ✅ 完全支持 | ❌ | ❌ | ❌ |
| 插件生态 | ✅ 2000+ | ⚠️ 有限 | ❌ | ⚠️ 有限 |
| 隐私安全 | ✅ 数据在本地 | ⚠️ 依赖服务商 | ⚠️ 依赖服务商 | ⚠️ 依赖服务商 |
| 定价 | **免费**（同步付费） | 免费/付费 | 免费/付费 | 付费 |

### 为什么产品经理适合用 Obsidian

- **需求文档、竞品分析、会议纪要**天然适合 Markdown
- **双向链接**帮助串联碎片化的产品思考
- **知识图谱**可视化呈现知识体系全貌
- **本地存储**保障敏感商业信息安全
- **高度可定制**适配不同团队和项目的工作流

## 2. 安装与初始设置

### 2.1 下载安装

- 官网下载：[https://obsidian.md](https://obsidian.md)
- 支持平台：Windows / macOS / Linux / iOS / Android

### 2.2 创建 Vault（仓库）

Vault 是 Obsidian 的核心概念，**一个 Vault = 一个本地文件夹 = 你的整个知识库**。

```
操作路径：打开 Obsidian → Create new vault → 选择存储位置 → 命名
```

**命名建议**：`PM-Knowledge-Base` 或 `产品知识库`

### 2.3 首次设置清单

| 设置项 | 推荐配置 | 路径 |
|--------|----------|------|
| 语言 | 中文 | 设置 → 关于 → 语言 |
| 默认编辑模式 | 实时预览（Live Preview） | 设置 → 编辑器 → 默认编辑模式 |
| 自动保存 | 开启（默认已开启） | 默认即可 |
| 附件存储位置 | 当前文件所在文件夹的子文件夹 | 设置 → 文件与链接 → 附件默认存放路径 |
| 新建笔记存放位置 | 指定的文件夹（Inbox） | 设置 → 文件与链接 → 新建笔记的存放位置 |

## 3. 核心概念速览

### 3.1 Markdown 基础

Obsidian 使用 Markdown 语法，以下是你需要掌握的最小集合：

```markdown
# 一级标题
## 二级标题
### 三级标题

**加粗文本**
*斜体文本*
~~删除线~~
==高亮文本==

- 无序列表
1. 有序列表
- [ ] 待办事项
- [x] 已完成事项

> 引用内容

| 表头1 | 表头2 |
|-------|-------|
| 内容1 | 内容2 |

`行内代码`

---   （分割线）
```

### 3.2 双向链接（Bi-directional Links）

这是 Obsidian 的**灵魂功能**。

```markdown
[[笔记名称]]          → 链接到另一篇笔记
[[笔记名称|显示文本]]  → 链接到另一篇笔记，但显示自定义文本
[[笔记名称#标题]]     → 链接到某篇笔记的具体标题
```

**使用场景举例**：
- 在需求文档中链接到 `[[竞品分析-抖音]]`
- 在会议纪要中链接到 `[[Q2 产品路线图]]`
- 在复盘笔记中链接到 `[[用户反馈汇总-2026Q1]]`

### 3.3 标签（Tags）

```markdown
#产品思考  #需求管理  #竞品分析
```

标签和双向链接的区别：
- **标签**：适合分类和快速筛选
- **双向链接**：适合建立知识之间的深层关联

### 3.4 知识图谱（Graph View）

快捷键 `Ctrl/Cmd + G` 打开全局图谱，可视化查看笔记之间的关联关系。

### 3.5 反向链接（Backlinks）

打开任何一篇笔记，右侧面板可查看**哪些笔记链接到了当前笔记**。这是发现知识关联的利器。

## 4. 基础操作入门

### 4.1 常用快捷键

| 操作 | Windows/Linux | macOS |
|------|--------------|-------|
| 新建笔记 | `Ctrl + N` | `Cmd + N` |
| 快速打开笔记 | `Ctrl + O` | `Cmd + O` |
| 命令面板 | `Ctrl + P` | `Cmd + P` |
| 搜索全库 | `Ctrl + Shift + F` | `Cmd + Shift + F` |
| 切换编辑/预览 | `Ctrl + E` | `Cmd + E` |
| 打开图谱 | `Ctrl + G` | `Cmd + G` |
| 分屏 | `Ctrl + \` | `Cmd + \` |
| 关闭当前标签页 | `Ctrl + W` | `Cmd + W` |

> **提示**：`Ctrl/Cmd + P` 打开命令面板是你的「万能遥控器」，几乎所有功能都可以在这里搜索并执行。

### 4.2 写笔记的流程

1. `Ctrl/Cmd + N` 新建笔记
2. 输入标题（即文件名）
3. 写入内容，用 Markdown 格式
4. 用 `[[]]` 链接相关笔记
5. 添加标签进行分类
6. 自动保存，无需手动操作

### 4.3 模板功能（核心模板）

在设置中启用「核心插件 → 模板」，指定模板文件夹后，即可快速插入预设模板。

**设置路径**：设置 → 核心插件 → 模板 → 开启 → 指定模板文件夹

## 7. 进阶用法

### 7.1 Dataview 查询示例

Dataview 插件让你把笔记变成可查询的数据库。

**查询所有进行中的需求**：

````markdown
```dataview
TABLE status, priority, created
FROM "01-Projects"
WHERE contains(tags, "需求") AND status != "已上线"
SORT priority ASC
```
````

**查询本周的会议纪要**：

````markdown
```dataview
LIST
FROM "06-Meeting"
WHERE date >= date(today) - dur(7 days)
SORT date DESC
```
````

**查询所有待办事项**：

````markdown
```dataview
TASK
FROM "01-Projects"
WHERE !completed
```
````

### 7.2 YAML Front Matter（属性）

每篇笔记顶部的 `---` 包裹区域是元数据，用于分类、筛选和 Dataview 查询。

```yaml
---
tags: [需求, 用户增长]
status: 进行中
priority: P1
owner: 张三
created: 2026-05-08
due: 2026-06-01
---
```

### 7.3 Callout 语法

Obsidian 内置 Callout 用于强调重要信息：

```markdown
> [!tip] 提示
> 这是一条提示信息

> [!warning] 注意
> 这是一条警告信息

> [!info] 信息
> 这是一条说明信息

> [!question] 问题
> 这是一个待解决的问题

> [!example] 示例
> 这是一个使用案例
```

### 7.4 嵌入笔记内容

```markdown
![[笔记名称]]           → 嵌入整篇笔记
![[笔记名称#标题]]      → 嵌入某个章节
![[笔记名称^block-id]]  → 嵌入某个段落
![[图片.png]]           → 嵌入图片
![[图片.png|300]]       → 嵌入图片并指定宽度
```

## 8. 搭配 AI（ima）使用技巧

### 8.1 整合思路

将 Obsidian 作为**知识沉淀层**，ima 作为**智能辅助层**：

```
信息输入 → ima 整理/分析 → Obsidian 沉淀 → 双向链接串联 → 知识复利
```

### 8.2 具体使用场景

#### 场景一：快速整理会议纪要

1. 会议中用 ima 实时记录或录音转文字
2. 让 ima 按你的会议纪要模板格式化内容
3. 粘贴到 Obsidian 对应笔记中
4. 添加双向链接关联到项目和需求文档

**给 ima 的提示词示例**：

```
请帮我将以下会议内容整理为结构化的会议纪要，格式如下：
## 议题与讨论
## 决策事项
## Action Items（表格形式：事项 | 负责人 | 截止时间）
以下是会议内容：
{粘贴内容}
```

#### 场景二：竞品分析辅助

1. 用 ima 搜集竞品信息并生成初步分析
2. 将结果粘贴到 Obsidian 的竞品分析模板中
3. 补充个人洞察和链接

**给 ima 的提示词示例**：

```
请帮我分析 {竞品名称} 这款产品，包含以下维度：
1. 产品定位与目标用户
2. 核心功能列表
3. 商业模式
4. 优劣势分析
5. 值得我们借鉴的地方
请用 Markdown 格式输出。
```

#### 场景三：需求文档起草

1. 将需求的初步想法告诉 ima
2. 让 ima 按模板格式生成需求文档初稿
3. 粘贴到 Obsidian，补充细节和链接

**给 ima 的提示词示例**：

```
我要写一个需求文档，需求简述如下：
{简要描述需求}
请帮我按以下结构生成需求文档：
- 背景与目标
- 用户故事
- 功能描述（核心流程 + 详细规则 + 异常处理）
- 数据埋点需求建议
用 Markdown 格式输出。
```

#### 场景四：知识总结与复盘

1. 将一段时间积累的笔记内容发给 ima
2. 让 ima 帮你提炼关键洞察和行动建议
3. 生成的总结存入 Obsidian 并关联原始笔记

### 8.3 高效协作工作流

```
日常工作 ──→ ima 辅助处理 ──→ Obsidian 沉淀
    ↑                              │
    └──── 知识检索与复用 ←──────────┘
```

**关键原则**：
- ima 负责**生成和加工**信息
- Obsidian 负责**存储和关联**知识
- 人（你）负责**判断和决策**

## 10. 常见问题与避坑指南

### Q1：笔记越来越多，找不到怎么办？

**解决方案**：善用以下方式检索
- `Ctrl/Cmd + O`：快速打开，按文件名搜索
- `Ctrl/Cmd + Shift + F`：全库搜索，搜索内容
- 使用标签系统分类
- 使用 Dataview 创建动态索引页

### Q2：要不要严格按照文件夹分类？

不必过于纠结文件夹结构。Obsidian 的设计理念是**以链接代替层级**。建议：
- 文件夹仅做粗粒度分类（5-8 个即可）
- 细粒度组织依靠**双向链接**和**标签**
- 用 MOC（Map of Content 内容地图）笔记做索引

### Q3：如何多端同步？

| 方案 | 优点 | 缺点 |
|------|------|------|
| Obsidian Sync（官方） | 稳定、端到端加密 | 付费（$4/月） |
| iCloud | 免费（Apple 生态） | 仅限 Apple 设备 |
| OneDrive | 免费额度大 | 偶有冲突 |
| Git | 免费、版本控制 | 设置门槛较高 |
| Remotely Save 插件 | 支持多种云存储 | 需要自行配置 |

### Q4：笔记命名有什么建议？

推荐命名规范：
- 需求文档：`PRD-{项目名}-{功能名}`
- 竞品分析：`竞品分析-{产品名}-{日期}`
- 会议纪要：`会议-{主题}-{日期}`
- 每日笔记：自动使用日期格式 `YYYY-MM-DD`
- 方法论：直接使用名称如 `用户故事地图`

### Q5：如何避免「工具焦虑」？

- **先用起来，再优化**。不要花太多时间研究完美的结构
- 第一周只用基础功能：新建笔记、双向链接、标签
- 第二周引入模板和每日笔记
- 第三周开始安装插件
- 持续迭代，让系统随你的需求进化
